# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take security seriously. If you discover a vulnerability in the SSPM Engine, please report it responsibly.

**Do NOT report security vulnerabilities via public GitHub issues.**

Instead, please report them via email to `security@sspm-engine.local`.

### What to Include

*   Description of the vulnerability.
*   Steps to reproduce the issue.
*   Potential impact.
*   Any proof-of-concept code (if available).

### Response Timeline

1.  **Acknowledgment**: We will acknowledge your report within 48 hours.
2.  **Assessment**: We will investigate the issue and determine its impact.
3.  **Fix**: We will work on a patch and release it as soon as possible.
4.  **Disclosure**: Once the fix is deployed, we will publicly disclose the vulnerability.
